# LLM Monitor Reports

Session statistics reports generated by [LLM Cockpit Monitor](https://github.com/jimdawdy-hub/llm-cockpit-monitor) — an aviation-themed LLM provider health dashboard.

## What these reports contain

Each report is a self-contained HTML file generated on demand from the live monitoring session. Open any file directly in a browser — no server required.

Each report includes:

- **Provider summary table** — probe count, success rate, average/P75/P95/max latency, error count
- **Latency over time chart** — per-provider average latency in 5-minute bins
- **Success rate over time chart** — per-provider success rate across the session window
- **AI analysis** — plain-language narrative written by DeepSeek, covering provider behaviour, patterns, incidents, and a comparative assessment
- **Executive summary** — one-paragraph bottom line

## Probe methodology

Each provider is tested with a live inference request (`1+1=`), not a ping. This distinguishes provider-side hangs from network loss and catches frozen connections that a simple health endpoint would miss.

## Report naming

Files are named `llm-cockpit-report-YYYYMMDD-HHMM.html` — the timestamp is when the report was generated.
